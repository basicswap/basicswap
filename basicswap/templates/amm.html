Offer Timing

  • min_seconds_between_offers: Minimum delay between creating offers (default: 15)
  • max_seconds_between_offers: Maximum delay between creating offers (default: 60)

Bid Timing

  • min_seconds_between_bids: Minimum delay between creating bids (default: 15)
  • max_seconds_between_bids: Maximum delay between creating bids (default: 60)

Script Behavior

  • main_loop_delay: Seconds between main loop iterations (10-1000)
  • auth: Basicswap API auth string, e.g., "admin:password". Ignored if client auth is not enabled.
  • wallet_port_override: If needed, uncomment and set to override wallet API port (for testing only)

State Management

  • prune_state_delay: Seconds between pruning old state data (0 to disable)
  • prune_state_after_seconds: How long to keep old state data (default: 7 days)

Automation Strategies

  • accept_all: Auto-accept all bids (default)
  • accept_known: Only accept bids from known identities
  • none: Manual acceptance required

Offer templates define how the AMM creates offers. Each template can have the following settings:

Required Settings

  • name: Template name, must be unique
  • enabled: Set to true to enable this offer
  • coin_from: Coin you send
  • coin_to: Coin you receive
  • amount: Amount to create the offer for
  • minrate: Rate below which the offer won't drop
  • min_coin_from_amt: Won't generate offers if wallet balance would drop below this (default: 0)
  • adjust_rates_based_on_market: Controls how AMM determines competitive rates. See detailed explanation below.

Optional Settings

  • ratetweakpercent: Modify the offer rate from the fetched value (can be negative)
  • amount_variable: Whether bidder can set a different amount
  • address: Address offer is sent from (auto = generate new address per offer)
  • offer_valid_seconds: How long generated offers will be valid for (minimum 600 seconds)
  • swap_type: Type of swap, defaults to "adaptor_sig"
  • min_swap_amount: Minimum purchase quantity when offer amount is variable
  • amount_step: Mandatory privacy feature that sets the offer size increment. Prevents revealing exact wallet balance by creating stepped offers. Must be >= 0.001 and <= your offer amount. Default: 0.001

Bid templates define how the AMM creates bids. Each template can have the following settings:

Required Settings

  • name: Template name, must be unique
  • enabled: Set to true to enable this bid
  • coin_from: Coin you receive
  • coin_to: Coin you send
  • amount: Amount to bid (minimum: 0.001)
  • max_rate: Maximum rate for bids
  • min_coin_to_balance: Won't send bids if wallet amount would drop below this

Optional Settings

  • max_concurrent: Maximum number of bids to have active at once (default: 1)
  • amount_variable: Can send bids below the set amount where possible
  • max_coin_from_balance: If needed, uncomment and set to limit bids when wallet amount is above this
  • address: Address bid is sent from (auto = generate new address per bid)
  • min_swap_amount: Minimum swap amount for variable amount bids
  • use_balance_bidding: If true, calculates bid amount as (wallet_balance - offer_min_amount) instead of using template amount
  • offers_to_bid_on: Filter offers to bid on: "auto_accept_only" (only auto-accept offers, default), "all" (any offer), "known_only" (known identities only)

The adjust_rates_based_on_market setting controls how your AMM determines competitive rates for offers. Here are all available options:

Coingecko

Value: false
Behavior: Uses only CoinGecko exchange rates. Fails if CoinGecko data is unavailable.
Best for: Stable, external market pricing without local market influence.

Orderbook

Value: "only"
Behavior: Uses only Orderbook offers that are as or more permissive than your automation_strategy. Fails if no matches.
Best for: Competing only with other market makers.

Either

Value: true
Behavior: Uses the higher rate between CoinGecko and market offers. Fails only if BOTH sources are unavailable.
Best for: Balanced approach with fallback protection.

Both

Value: "all"
Behavior: Uses the higher rate between CoinGecko and market offers. Fails if EITHER source is unavailable.
Best for: Strict validation requiring both external and market data.

Orderbook - Fallback

Value: "minrate"
Behavior: Uses market offers if available, otherwise falls back to your minimum rate.
Best for: Market-responsive pricing with guaranteed operation.

Static

Value: "static"
Behavior: Uses only your minimum rate + rate tweak percentage. Ignores all external data.
Best for: Fixed pricing strategies or testing.

Rate Calculation Formula

Final Rate = Base Rate × (ratetweakpercent / 100 + 1)
Where Base Rate comes from your selected adjustment method above.

Important Notes
  • • Final rate will never go below your minrate setting
  • • Strategies which use the orderbook will only consider offers with an automation strategy as permissive or better than your config
  • • If your chosen method fails to get rates, the offer will be skipped for that cycle (Exception: "Static" and "Orderbook (Fallback)")
  • • Rate tweak percentage can be negative to offer below market rates

Attempt Bids First allows your AMM to fill existing offers before creating new ones.

How it works

1. AMM wants to create a 1 BTC → XMR offer at rate 150
2. Checks for existing XMR → BTC offers that would fill our intended offer
3. Places bids on compatible offers to fill what we can
4. Creates AMM offer only for the remaining amount

Bidding Strategies
Aggressive: Bid on all compatible offers to fill as much as possible
Balanced: Mix of bidding and offer creation (default)
Conservative: Only bid on offers that match your automation strategy
Best rates from auto-accept offers only: Only bid on offers that match your automation strategy, prioritizing best rates
Automation Strategy Compatibility

Bidding strategies consider your AMM's automation strategy when selecting offers to bid on:
• Accept All: Only bids on offers with "Accept All" automation
• Accept Known: Bids on offers with "Accept All" or "Accept Known" automation
• None: Bids on all offers regardless of automation strategy

Configuration Options
Max Bid Percentage: Maximum % of your offer amount to use for bidding (0-100)
Rate Tolerance %: Bid on offers up to this % better than your intended rate
Min Remaining Offer: Don't create AMM offer if remainder is below this amount
